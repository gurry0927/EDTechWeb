# Token 成本分析與優化方案 (Cost Control)

針對「一人公司」的初期投入，我們必須精算每一塊錢。以下是如何用「一杯大冰奶」的月預算，跑起這套 AI 自動化流水線的策略。

## 1. 成本估算 (以 GPT-4o-mini 為例)

目前如 `GPT-4o-mini` 的價格大約是 **$0.15 / 100 萬 Input Tokens**。

| 項目 | 數量 | Token 估算 | 每日成本 (USD) |
| :--- | :--- | :--- | :--- |
| **新聞標題初選 (Scoring)** | 100 則/日 | ~5,000 | $0.00075 |
| **一週精選生成 (Drafting)** | 1 則/週 | ~2,000 | $0.0003 (週) |
| **總計 (每月流量)** | | ~16 萬 | **約 $0.024 (約不到 1 元台幣)** |

**結論**：如果你用的是中低階模型進行「評分」，成本幾乎可以忽略不計。

## 2. 核心省錢策略 (Optimization)

### A. 階層式過濾 (Tiered Filtering) - 0 成本攔截
- **第一層：程式過濾 (Regex/Keyword)**：用關鍵字清單（如：行政、預算、衝突）過濾掉 80% 無關新聞。這在 Python 裡跑是 **0 成本**。
- **第二層：低階 AI 評分 (Cheap Model)**：只有關鍵字中的新聞才餵給 `GPT-4o-mini`。
- **第三層：高階 AI 生成 (Pro Model)**：只有最後你選中的那一則，才用 `GPT-4o` 或 `Claude 3.5 Sonnet` 生成精美文案。

### B. 批次處理 (Batch Processing)
- 不要一條新聞發一次 Request。一次餵 20 個標題，讓 AI 回傳一個 JSON List 包含 20 個評分。這樣可以共用系統提示 (System Prompt)，大幅節省 Token。

### C. 短文本處理 (Title & Snippet Only)
- 評分不需要看全文。只需要看「新聞標題 + 摘要」，這已經足夠判定教育價值。

## 3. 前期投入預估 (First 3 Months)

- **API 儲值金**：儲 5 美元 (約 160 台幣) 大概可以讓你跑一年，除非你的流量暴增。
- **雲端成本**：託管在 Vercel 是免費的；Supabase 的免費額度對初期來說綽綽有餘。

## 4. 總結
這套方案的「人工成本」（你的時間）遠大於「Token 成本」。
如果你決定開始實作，我們先從 **「0 成本關鍵字過濾」** 開始寫，等到真的需要 AI 判斷時，再接上 API。
