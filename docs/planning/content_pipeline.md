# AI Content Pipeline: 自動化教育新聞萃取方案

針對「如何判定有意義的新聞」以及「減少人工審核負擔」，我們建議採用 **「三階過濾法」**。

## 1. 方案架構：從海量資訊到教育精華

### 第一階：來源採集 (Data Collection)
- **來源**：Google News RSS, 中央社專欄, 或特定公部門公告。
- **初篩關鍵字**：輸入「地方政府」、「行政區」、「國土規劃」、「環評」、「預算」等 108 課綱關鍵字。

### 第二階：AI 教育價值評分 (AI Scoring) - 解決「意義觀」問題
這一步是核心。我們不讓 AI 「找新聞」，而是讓它 **「當老師」**。
- **LLM Agent 任務**：給予 AI 完整的 108 課綱社會科指標 (Index)，讓它評估：
    - 「這則新聞是否能對應到特定課綱單元？」 (Correlativity)
    - 「這則新聞是否包含可客觀判讀的數據或地理位置？」 (Factuality)
    - 「這則新聞是否存在爭議點可引起思辨？」 (Debatability)
- **結果**：AI 回傳 1-10 分，系統僅提取 8 分以上的項目進入後選區。

### 第三階：自動草稿生成 (Auto-Drafting) - 解決「科學審核」問題
這一步將「審核」變成「選擇題」。
- **生成內容**：AI 直接根據 8 分以上的新聞，生成 JSON 格式的數據：
    - 地圖標註點 (Latitude, Longitude)
    - 故事標題與 200 字導讀
    - 3 個蘇格拉底式引導問題
- **人工動作**：你每週只需打開 Admin Dashboard，看到 3 則 AI 推薦的「本週精選」，你只需要點擊 **[發布]** 或 **[微調文案]**。

## 2. 實作建議 (Tech Implementation)

- **工具選擇**：
    - **Scraper**: `Python + BeautifulSoup` 或簡單的 `Next.js API routes + rss-parser`。
    - **Logic**: 透過 OpenAI 或 Claude 的 API 進行內容評分與生成。
    - **Scheduler**: 使用 `GitHub Actions` (免費且強大) 或 `Vercel Cron` 每週一凌晨執行一次。

## 3. 「有意義」的定義範例

| 新聞類型 | AI 評分 (1-10) | 理由 |
| :--- | :--- | :--- |
| 「某縣市推動新的垃圾焚化爐選址爭議」 | 9 | 符合地方自治、外部性、公共參與等課綱。 |
| 「某明星在台北開演唱會」 | 2 | 無直接教育與地理素養關聯。 |
| 「行政院通過基隆捷運規劃案」 | 8 | 符合區域發展、預算分配、交通建設等概念。 |
